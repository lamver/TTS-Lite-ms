# Piper TTS Микросервис (FastAPI + Piper + RabbitMQ)

Производственный микросервис для синтеза речи, построенный на стеке **FastAPI**, **Piper TTS** и **RabbitMQ**. Предоставляет как HTTP API для синтеза по запросу, так и асинхронную обработку задач через очередь сообщений с возможностью сохранения результатов в S3-совместимое хранилище.

## Архитектура

Сервис сочетает в себе несколько ключевых технологий:

- **FastAPI** – современный веб-фреймворк для предоставления HTTP API с автоматической документацией (Swagger/OpenAPI).
- **Piper** – высококачественный локальный синтезатор речи с поддержкой ONNX-моделей и низкой задержкой.
- **RabbitMQ** – брокер сообщений для организации очередей задач, обеспечивающий масштабируемость и отказоустойчивость.
- **S3/MinIO** – совместимое объектное хранилище для долговременного хранения сгенерированных аудиофайлов.
- **Docker** – контейнеризация для простого развертывания в любом окружении.

## Возможности

### 1. HTTP API (синхронный режим)

- **`GET /models`** – возвращает полный список доступных моделей с метаданными (спикеры, частота дискретизации, примеры).
- **`GET /generate`** – синтезирует речь из переданного текста с настройками модели, спикера, скорости и уровня шума. Возвращает готовый WAV-файл.
- **`GET /get-audio/{filename}`** – отдает ранее сгенерированный файл из временной папки `outputs/` с автоматическим удалением после отправки.

### 2. Обработка очереди (асинхронный режим)

- **Входящая очередь** (`text_to_voice_lite_piter`) – принимает задачи в формате JSON.
- **Фоновая задача** (`process_mq_tasks`) – непрерывно обрабатывает сообщения, выполняет синтез, загружает результат в S3 и отправляет подтверждение в **исходящую очередь** (`text_to_voice_lite_piter_result`).
- **Отчет об ошибках** – при неудаче в исходящую очередь отправляется детализированное сообщение об ошибке.

### 3. Управление моделями

- **Автоматическое сканирование** – при старте сервис рекурсивно ищет файлы `.onnx` и `.onnx.json` в директории `piper_models/`.
- **Предзагрузка** – модель, указанная в `DEFAULT_MODEL`, загружается в память сразу, что ускоряет первый запрос.
- **Динамическая загрузка** – остальные модели подгружаются по требованию (ленивая загрузка) и кэшируются в глобальном словаре `loaded_voices`.

### 4. Интеграция с S3

- **Конфигурация через переменные окружения** – endpoint, ключи доступа, имя бакета.
- **Метаданные** – при загрузке файла в S3 добавляется метаданное с укороченным текстом (первые 100 символов).
- **Автоочистка** – локальный файл удаляется сразу после успешной загрузки в облако.

## Быстрый старт

### 1. Клонирование и настройка окружения

```bash
git clone <репозиторий>
cd <папка проекта>
cp .env.example .env
```

Отредактируйте `.env`, указав актуальные параметры RabbitMQ и S3 (если требуется).

### 2. Загрузка моделей Piper

Используйте готовый скрипт:

```bash
bash download_models.sh
```

Или вручную скачайте нужные модели с [официального репозитория](https://github.com/rhasspy/piper) и разместите их в `piper_models/` (сохраняя структуру подпапок).

### 3. Установка зависимостей

```bash
pip install -r requirements.txt
```

### 4. Запуск сервиса

**Локально (для разработки):**

```bash
python main.py
```

Сервер будет доступен по адресу `http://localhost:8000`. Документация API – `http://localhost:8000/docs`.

**Через Docker (продакшен):**

```bash
docker-compose up --build
```

## Конфигурация окружения

Все настройки вынесены в переменные окружения. Основные параметры:

| Переменная | Описание | Пример |
|------------|----------|--------|
| `RABBITMQ_URL` | URL подключения к RabbitMQ | `amqp://guest:guest@rabbit:5672/` |
| `QUEUE_NAME` | Имя входящей очереди | `text_to_voice_lite_piter` |
| `S3_ENDPOINT` | Endpoint S3-совместимого хранилища | `http://minio:9000` |
| `S3_ACCESS_KEY` | Ключ доступа S3 | `minioadmin` |
| `S3_SECRET_KEY` | Секретный ключ S3 | `minioadmin` |
| `S3_BUCKET` | Имя бакета | `tts-audio` |
| `DEFAULT_MODEL` | Модель, загружаемая при старте | `ru_RU-denis-medium` |
| `MODELS_DIR` | Директория с моделями (опционально) | `piper_models` |

## Примеры использования

### Через HTTP API

```bash
# Получить список моделей
curl http://localhost:8000/models | jq .

# Сгенерировать аудио
curl -o hello.wav "http://localhost:8000/generate?text=Привет, мир!&model=ru_RU-denis-medium&speaker=0&speed=1.2"
```

### Через RabbitMQ

Отправьте сообщение в очередь `text_to_voice_lite_piter`:

```json
{
  "requestId": "req-12345",
  "text": "Это тестовый текст для синтеза речи.",
  "model": "ru_RU-denis-medium",
  "speaker": "0"
}
```

После обработки в очередь `text_to_voice_lite_piter_result` придет ответ:

```json
{
  "requestId": "req-12345",
  "status": true,
  "path": "WORKERS/text_to_voice_lite_piter/req-12345/req-12345.wav"
}
```

## Разработка и расширение

### Добавление новой модели

1. Поместите файлы модели (`*.onnx` и `*.onnx.json`) в поддиректорию `piper_models/`.
2. Перезапустите сервис – модель автоматически появится в реестре `models_registry`.
3. Используйте её идентификатор (имя файла без `.onnx`) в запросах.

### Кастомизация параметров синтеза

В HTTP-эндпоинте `/generate` доступны параметры:

- `speed` (`float`) – коэффициент скорости произношения (по умолчанию 1.0).
- `noise` (`float`) – уровень шума (по умолчанию 0.667).

В сообщениях RabbitMQ эти параметры пока не поддерживаются, но их можно добавить, расширив логику `process_mq_tasks`.

### Мониторинг

- **Журналирование** – все ключевые события выводятся в stdout с пометками `--- ... ---`.
- **Метрики здоровья** – можно добавить эндпоинт `/health` (пока отсутствует).
- **Отслеживание очередей** – используйте CLI RabbitMQ: `rabbitmqctl list_queues`.
